# CalorieEstimator 모듈

## 1. 기능 요약

CalorieEstimator 모듈은 사용자의 신체정보와 러닝 거리를 기반으로 예상 칼로리 소모량을 계산하는 기능이다.

또한 계산된 칼로리를 음식 기준으로 환산하여 사용자가 보다 직관적으로 이해할 수 있도록 제공한다.

예를 들어 "500kcal 소모" 대신 "햄버거 1개 분량 소모"와 같이 표시하여 운동 효과를 쉽게 확인할 수 있다.

---

## 2. 사용 자료구조 / 알고리즘

### 사용 자료구조

#### Number

사용자 신체정보와 칼로리 계산 결과를 저장하기 위해 사용하였다.

저장 데이터

- 나이(age)
- 키(height)
- 체중(weight)
- 칼로리(calories)

주요 연산

- 곱셈
- 나눗셈
- 반올림

선택 이유

칼로리 계산은 수치 기반 계산이므로 Number 자료형이 적합하다.

---

#### React State(useState)

사용자가 입력하는 정보를 실시간으로 저장하기 위해 사용하였다.

관리 데이터

- 성별
- 나이
- 키
- 체중

선택 이유

입력 값 변경 시 즉시 계산 결과를 화면에 반영하기 위함이다.

---

### 사용 알고리즘

#### MET 기반 칼로리 추정 알고리즘

설명

러닝 시 일반적으로 사용되는 에너지 소비량 공식을 활용하여 예상 칼로리를 계산한다.

공식

칼로리 = 거리 × 체중 × 1.036

예시

- 거리 : 5km
- 체중 : 70kg

결과

- 5 × 70 × 1.036
- 약 363kcal

선택 이유

실제 러닝 칼로리 계산에 널리 사용되는 근사식이며 계산이 간단하고 결과 해석이 쉽다.

---

#### Food Equivalent Conversion Algorithm

설명

계산된 칼로리를 음식 칼로리 기준으로 환산하는 알고리즘이다.

예시

- 밥 1공기 : 300kcal
- 햄버거 1개 : 550kcal
- 피자 1조각 : 285kcal
- 콜라 1캔 : 140kcal

환산 예시

500kcal 소모 시

- 밥 약 1.7공기
- 햄버거 약 0.9개
- 피자 약 1.8조각
- 콜라 약 3.6캔

선택 이유

사용자가 숫자만 보는 것보다 운동 효과를 직관적으로 이해할 수 있다.

---
