모듈 6. shapeAccuracy.js ( frontend/src/utils/shapeAccuracy.js )

6-1) calculateShapeAccuracy()

* 역할 : 사용자가 선택한 Shape 경로와 실제 도로 기반으로 생성된 경로를 비교하여 Shape 유사도를 점수화

* 사용 알고리즘 : 최근접 거리 기반 Shape Similarity 알고리즘 (Nearest Distance Matching)

ShapeRunner는 사용자가 선택한 도형(하트, 별, 뼈 등)을 실제 도로망에 맞게 변환하는 과정에서 원본 Shape와 차이가 발생한다.

따라서 이상적인 Shape 좌표(route)와 실제 생성된 경로(fullPathCoordinates)를 비교하여 도형이 얼마나 정확하게 유지되었는지 정량적으로 평가할 필요가 있었다.

본 알고리즘은 원본 Shape의 각 좌표에 대해 실제 경로상의 가장 가까운 좌표를 탐색한 뒤 평균 오차를 계산한다. 평균 오차가 작을수록 Shape가 잘 유지된 것으로 판단하며 이를 0~100점 범위의 Accuracy 점수로 변환한다.

현재 프로젝트 규모에서는 구현이 단순하고 계산 비용이 낮으며 충분한 성능을 보이기 때문에 선택하였다.

* 사용 자료구조 :

10. 배열(Array)

route, fullPathCoordinates 좌표 데이터를 순차적으로 비교해야 하기 때문에 배열을 사용하였다.

좌표 순서를 유지하면서 반복 탐색을 수행하기에 적합하였다.

11. 최소 거리 추적 변수(Min Distance Tracking)

각 Shape 좌표마다 가장 가까운 실제 경로 좌표를 탐색하기 위해 최소 거리 값을 저장하였다.

최근접 좌표 탐색 과정에서 최솟값을 효율적으로 갱신할 수 있기 때문에 사용하였다.
